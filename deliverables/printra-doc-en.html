Printra Customer Delivery Documentation (English)

Version: 0.1.5 | Date: March 25, 2026 | Document Type: End-to-End Technical + Operational

Monorepo Web + API + Worker Studio + Mockup Automatic Analysis + Chrome Extension

1. Project Overview

Printra is a SaaS core that unifies design, mockup workflows, competitor analysis, content generation, project management, admin/billing, and platform operations in one shell. The system follows a "customer-owned infrastructure" delivery model: real customer credentials are not hardcoded during development, and final live activation is completed on customer infrastructure.

3. Category-by-Category Features

3.1 Dashboard

  • Live health, storage readiness, extension status, and latest analysis snapshots.
  • API sources: GET /health, GET /automatic-analysis/history, GET /automatic-analysis/extension-status, GET /storage/foundation.
  • Operational handoff status for customer deployment.

3.2 Projects

  • Create, update, duplicate, favorite, archive, and delete project workspaces.
  • Bidirectional synchronization with Studio via project id.
  • API: /projects CRUD + /duplicate + /studio-state.
  • Local foundation file: apps/api/data/projects.json.

3.3 Studio

  • Canva-style editing, mockup stage management, design-to-mockup projection.
  • Object types: text, image, shape, mockup. Drag/resize/rotate enabled.
  • Projection presets: flat-front, relaxed-front, oversize-front, hoodie-front, sleeve-badge.
  • Library drawer tabs: Mockups / Designs / Tools.
  • Clear-scene action available for full reset.
  • Export flow: server-side export job + worker ZIP generation.

3.4 Library

  • Section management (mockup/design), asset list, duplicate, delete, upload.
  • Two storage operating modes:
    • Local JSON Foundation: safe fallback when DB is not configured.
    • Database + R2: live mode when DATABASE_URL and real R2 credentials are provided.
  • API: /storage/foundation, /storage/sections, /storage/assets, /storage/uploads/prepare, /storage/uploads/finalize.

3.5 Analytics

  • Snapshot history view for Automatic Analysis outputs.
  • Foundation ready for batch comparison trends.

3.6 Automatic Analysis (Competitor Analysis)

  • Modes: URL, paste, manual, CSV.
  • Scores: listingStrength, storeStrength, keywordStrength, trustScore, competitionDifficulty, opportunityScore, conversionQuality, demandHeat.
  • Produces risk flags, opportunity notes, and beat map.
  • API: POST /automatic-analysis/analyze, POST /automatic-analysis/save, GET /automatic-analysis/history.
  • Extension handshake: /automatic-analysis/extension-status and /automatic-analysis/extension-ingest.

3.7 Create a List

  • SEO + GEO + AEO focused title/description/keyword generation.
  • 13-keyword rule set, scoring and improvement notes.
  • Default rule-based mode; AI-assisted mode when OPENAI_API_KEY is configured.
  • API: GET /market-intelligence/foundation, POST /market-intelligence/create-list.

3.8 Settings

  • Profile/workspace, localization, appearance, and studio default preferences.
  • API: GET /settings, PUT /settings, POST /settings/reset.

3.9 Billing

  • Plan selection, billing cycle, seat count, auto-renew, charge preview, usage, invoices.
  • API: GET /billing, PUT /billing, POST /billing/reset.
  • Local foundation file: apps/api/data/billing.json.

3.10 Admin

  • User matrix, role schema, plan/access policies, invitations, audit stream.
  • API: GET /admin, PUT /admin, POST /admin/reset.
  • Local foundation file: apps/api/data/admin.json.

3.11 Platform

  • System health board, connector readiness, extension heartbeat.
  • API: /health, /connectors/foundation, /automatic-analysis/extension-status.

4. Core Working Systems (Flows)

4.1 Studio Persistence + Autosave

  • Local draft persistence: IndexedDB (printra-studio) + runtime snapshot.
  • Project-scoped keys via project:{projectId}.
  • Local autosave debounce: 35ms.
  • Server autosave minimum interval: 3000ms.
  • Hash-based panel routing: #mockups, #designs, #tools.
  • Safe route flush hook: window.__printraFlushStudioDraft.

4.2 Library Storage Mode

  • System remains operational without DB via local-json foundation.
  • When DB + R2 are configured, signed upload/finalize is enabled.
  • Missing customer credentials return controlled readiness errors, not crashes.

4.3 Automatic Analysis + Extension Flow

  • Content script parses Etsy page, background script posts data to API.
  • Extension status TTL is approximately 15 minutes.
  • Ingest triggers analysis engine and optional snapshot save.

4.4 Export Pipeline

  • Web submits export payload (POST /exports).
  • API writes queue and status job records.
  • Worker consumes queue, produces ZIP, updates status to completed.
  • Download endpoint: GET /exports/{jobId}/download.

8. Vercel Deployment + Domain Setup

8.1 Web Deploy (from monorepo)

cd /d C:\Users\etsy2\OneDrive\Belgeler\New project\analysis-unzip
pnpm dlx vercel@latest login
pnpm dlx vercel@latest --cwd apps/web
pnpm dlx vercel@latest --cwd apps/web --prod

8.2 Vercel Environment Variables (Web project)

  • NEXT_PUBLIC_API_BASE_URL=https://api.your-domain.com
  • NEXT_PUBLIC_DEFAULT_ORGANIZATION_ID=...
  • Extension store URLs (optional but recommended)

8.3 Add Domains

pnpm dlx vercel@latest domains add your-domain.com
pnpm dlx vercel@latest domains add www.your-domain.com
pnpm dlx vercel@latest domains inspect your-domain.com

Apply the A/CNAME records provided by Vercel in your DNS panel.

8.4 API/Worker Deployment Note

In this repository, API is an Express service and Worker is a long-running process. For production, deploy API + Worker on a separate Node runtime (VPS / Render / Railway / Fly.io, etc.). Keep Web on Vercel and point it to your API host via NEXT_PUBLIC_API_BASE_URL.

9. Chrome Extension (Required for Competitor Analysis)

  • Extension path: apps/extension
  • Manifest version: v3
  • Host permissions include Etsy + local API/Web hosts
  • API Base can be set from extension popup

9.1 Customer Setup (Development/Test)

  1. Open Chrome chrome://extensions.
  2. Enable Developer mode.
  3. Use Load unpacked and select extension folder.
  4. On Etsy listing/shop pages, use "Analyze" / "Sync status" actions.

9.2 Production Note

When moving to customer live domain, update manifest.json host permissions to match production API/Web URLs.

10. Recommended Development Roadmap

Phase 1 - Performance and Stability

  • Studio render pipeline profiling for heavy mockup/design scenes.
  • Virtualization and incremental hydration strategies.
  • Autosave conflict telemetry and traceability.

Phase 2 - Infrastructure Hardening

  • Versioned DB migration management.
  • Live R2 upload verification + thumbnail generation pipeline.
  • API auth, rate limiting, and extended audit controls.

Phase 3 - Production Operations

  • Centralized logging, metrics, and alerting.
  • Backup/restore policy.
  • SLA and incident runbook.
