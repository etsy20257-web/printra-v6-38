Printra Müşteri Teslim Dokümantasyonu (Türkçe)

Sürüm: 0.1.5 | Tarih: 25 Mart 2026 | Belge tipi: Uçtan Uca Teknik + Operasyonel

Monorepo Web + API + Worker Studio + Mockup Automatic Analysis + Chrome Extension

1. Proje Özeti

Printra; tasarım, mockup, rakip analizi, içerik üretimi, proje yönetimi, admin/billing ve platform operasyonlarını tek panelde birleştiren bir SaaS çekirdeğidir. Sistem "müşteri sahipli altyapı" modeline göre hazırlanmıştır: geliştirme aşamasında gerçek müşteri servis hesapları kod içine gömülmez, canlı aktivasyon müşteri altyapısı ile yapılır.

2. Mimari ve Bileşenler

3. Kategori Bazlı Özellikler

3.1 Dashboard

  • Canlı sağlık verisi, storage readiness, extension durumu, son analiz snapshot’ları.
  • API kaynakları: GET /health, GET /automatic-analysis/history, GET /automatic-analysis/extension-status, GET /storage/foundation.
  • Müşteriye yönelik handoff modelinin anlık görünümü.

3.2 Projects

  • Proje oluşturma, güncelleme, çoğaltma, favori, arşivleme, silme.
  • Studio ile proje-id seviyesinde senkron çalışır.
  • API: /projects altında CRUD + /duplicate + /studio-state.
  • Yerel temel veri kaydı: apps/api/data/projects.json.

3.3 Studio

  • Canva-benzeri düzenleme, mockup stage yönetimi, design/multi-stage çalışma.
  • Objeler: text, image, shape, mockup. Drag/resize/rotate aktif.
  • Mockup projection presetleri: flat-front, relaxed-front, oversize-front, hoodie-front, sleeve-badge.
  • Library drawer: Mockups / Designs / Tools paneli.
  • Tüm sahneyi temizleme butonu (clear scene) mevcut.
  • Export akışı: server-side export job + worker ZIP üretimi.

3.4 Library

  • Section yönetimi (mockup/design), asset listeleme, çoğaltma, silme, yükleme.
  • Storage modu iki katmanlıdır:
    • Local JSON Foundation: DB yokken güvenli fallback.
    • Database + R2: DATABASE_URL ve gerçek R2 bilgileri geldiğinde canlı mod.
  • API: /storage/foundation, /storage/sections, /storage/assets, /storage/uploads/prepare, /storage/uploads/finalize.

3.5 Analytics

  • Automatic Analysis sonuç geçmişinin snapshot bazlı görünümü.
  • Batch karşılaştırma için temel veri yapısı hazır.

3.6 Automatic Analysis (Rakip Analizi)

  • Modlar: URL, paste, manual, CSV.
  • Skorlar: listingStrength, storeStrength, keywordStrength, trustScore, competitionDifficulty, opportunityScore, conversionQuality, demandHeat.
  • Risk flag, opportunity notes, beat map üretir.
  • API: POST /automatic-analysis/analyze, POST /automatic-analysis/save, GET /automatic-analysis/history.
  • Extension handshake: /automatic-analysis/extension-status ve /automatic-analysis/extension-ingest.

3.7 Create a List

  • SEO + GEO + AEO odaklı başlık/açıklama/keyword üretimi.
  • 13 anahtar kelime kuralı, kalite skoru ve iyileştirme notları.
  • Varsayılan rule-based; OPENAI_API_KEY varsa AI-assisted moda geçebilir.
  • API: GET /market-intelligence/foundation, POST /market-intelligence/create-list.

3.8 Settings

  • Profil/workspace, lokalizasyon, görünüm, studio default ayarları.
  • API: GET /settings, PUT /settings, POST /settings/reset.

3.9 Billing

  • Plan seçimi, cycle, seat, auto renew, charge preview, usage, invoices.
  • API: GET /billing, PUT /billing, POST /billing/reset.
  • Yerel temel veri: apps/api/data/billing.json.

3.10 Admin

  • Kullanıcı matrisi, rol şeması, plan/access policy, davetler, audit.
  • API: GET /admin, PUT /admin, POST /admin/reset.
  • Yerel temel veri: apps/api/data/admin.json.

3.11 Platform

  • Sistem health board, connector readiness (Google Drive/OneDrive), extension heartbeat.
  • API: /health, /connectors/foundation, /automatic-analysis/extension-status.

4. Çalışma Sistemleri (İş Akışları)

4.1 Studio Persist + Autosave Mimarisi

  • Local draft: IndexedDB (printra-studio) + runtime snapshot.
  • Project scoped draft key: project:{projectId} mantığı.
  • Local autosave debounce: 35ms.
  • Server autosave minimum aralık: 3000ms.
  • Hash/route tabanlı panel açma: #mockups, #designs, #tools.
  • Panel/route geçişinde flush mekanizması: window.__printraFlushStudioDraft.

4.2 Library Storage Modu

  • DB yoksa sistem kapanmaz; local-json foundation ile devam eder.
  • DB + R2 aktifse signed upload ve finalize akışı devreye girer.
  • Müşteri credential’ı yoksa API kontrollü 503 + açıklayıcı not döndürür (crash yerine).

4.3 Automatic Analysis + Extension Akışı

  • Content script Etsy sayfasını okur, background script API’ye iletir.
  • Extension status TTL: yaklaşık 15 dakika.
  • Ingest sonrası analiz motoru çalışır, istenirse snapshot kaydedilir.

4.4 Export Sistemi

  • Web, export payload’ı API’ye atar (POST /exports).
  • API queue/status dosyası üretir.
  • Worker queue’yu alır, ZIP üretir, status’u completed yapar.
  • Web indirme endpointi: GET /exports/{jobId}/download.

8.4 API/Worker Yayın Notu

Bu repoda API Express servisidir, worker ise sürekli çalışan process’tir. Üretimde API + worker için ayrı Node runtime (VPS / Render / Railway / Fly.io vb.) önerilir. Web tarafı Vercel’de, API tarafı ayrı hostta çalıştırılıp NEXT_PUBLIC_API_BASE_URL ile bağlanır.
